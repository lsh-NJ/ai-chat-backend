# 错误处理

## 1. 空消息

发送：

```
{
  "message": "     "
}
```

预期：

```
422 Unprocessable Entity
```

---

## 2. 不存在的会话

发送：

```
{
  "message": "你好",
  "conversation_id": 999999
}
```

预期：

```
{
  "detail": "conversation_id not found"
}
```

状态码：

```
404
```

---

## 3. 普通模型请求错误

普通 `/chat` 仍然会处理：

```
API Key 错误      → 401 / 403
模型服务异常      → 对应服务状态码
连接失败          → 502
超时              → 504
响应格式异常      → 502
```

---

## 4. 流式输出中途失败

如果模型已经开始输出，HTTP 响应状态码通常已经发给客户端，后端无法再把状态从 `200`​ 改成 `502`。

因此中途失败时：

```
[流式响应中断：stream interrupted: ReadTimeout]
```

已经生成的部分仍然会保存到数据库。
